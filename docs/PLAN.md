# 《字字成章》工作計畫

## 需求與範圍

建立可直接在瀏覽器執行的純靜態交錯式成語填空遊戲。首版包含三種難度、固定交錯成語盤、候選字、提示、計分、結算、偏好儲存、鍵盤與觸控操作，以及教育部《成語典》外部查詢連結。

不包含帳號、後端、金流、執行時擷取教育部網站、任意盤面生成、PWA 與多人遊戲。

## 里程碑與工作分解

1. 建立專案骨架、Node.js 內建測試、題庫資料與題組驗證。
2. 以測試先行完成盤面模型、抽題、填答、提示、計分與偏好降級服務。
3. 建立語意化 HTML、昭和益智節目 × 和紙書房視覺系統與三組響應式斷點。
4. 串接滑鼠、觸控、鍵盤、音效、儲存、結算與外部查詢連結。
5. 補齊 README、視覺方向、測試計畫、貢獻指南、MIT 授權與 GitHub Pages 部署檔案。
6. 執行自動測試、題庫驗證、HTTP 本機啟動、瀏覽器、行動裝置與無障礙測試。

## 風險與因應

- 題組正確性：用自動測試驗證交叉字、越界與難度資料。
- 手機版面：固定盤面尺寸與等比縮放，主要觸控區至少 44×44 CSS 像素。
- localStorage、音效或外部網站不可用：捕捉錯誤並降級，核心遊戲繼續執行。
- 著作權：僅內建成語與自撰提示，外部查詢連至教育部官方網站，不複製釋義。

## 驗收條件

- 三種難度皆能開局、填答、提示、重設、結算與再玩。
- 交錯盤交叉字一致；候選字、分數、提示與完成判定正確。
- 桌機、平板、手機可操作，鍵盤可完成一局，減少動態、可見焦點與對比度符合規格。
- `npm test`、題庫驗證與瀏覽器測試均有通過證據。
- 文件、MIT 授權與 GitHub Pages 部署說明完整，且提交未包含敏感檔案。
