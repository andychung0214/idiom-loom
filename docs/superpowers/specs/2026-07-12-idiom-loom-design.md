# 《字字成章》遊戲設計規格

## 產品定位

- 英文名稱：Idiom Loom
- 中文名稱：字字成章
- 類型：交錯式成語填空益智遊戲
- 平台：桌機瀏覽器優先，支援手機與平板觸控
- 首版部署：純靜態網站，可部署至 GitHub Pages 或 Synology NAS Web Station
- 技術：HTML、CSS、Vanilla JavaScript、原生 ES Modules、Web Audio API、localStorage

《字字成章》將電視益智節目的限時作答感，轉化為可獨立遊玩的交錯成語盤。玩家在橫直排列的四字成語中補上缺字，完成整盤後回顧成語並前往教育部《成語典》查詢。

## 成語資料與授權界線

- 首版使用專案內建、人工整理的四字成語題庫，不在瀏覽器執行期間擷取教育部網站。
- 題庫只收錄成語、難度、自撰提示與查詢關鍵字，不大量複製教育部《成語典》的釋義、典故或例句。
- 結算畫面以查詢網址連至教育部《成語典》，並清楚標示外部資料來源。
- 遊戲原始程式碼採 MIT License；第三方網站內容與商標不包含在專案授權範圍內。

## 核心遊戲循環

1. 玩家在標題畫面選擇入門、熟手或博學難度。
2. 系統從符合難度的內建題組選出一盤，呈現多組橫直交錯的四字成語。
3. 玩家點選空白字格，再從候選字盤選字；亦可使用鍵盤完成相同操作。
4. 正確答案立即鎖定並計分；錯誤答案不寫入，短暫顯示錯誤回饋。
5. 玩家可消耗有限提示次數，自動揭示目前選取字格。
6. 全部字格完成後進入結算，顯示分數、用時、正確率與本局成語清單。
7. 玩家可查詢個別成語、再玩同難度題組或返回選擇難度。

## 難度規則

| 難度 | 每組空白 | 提示 | 候選字干擾 | 適用情境 |
|---|---:|---:|---:|---|
| 入門 | 1–2 格 | 3 次 | 少量 | 初次遊玩與行動裝置 |
| 熟手 | 2–3 格 | 2 次 | 中量 | 一般遊玩 |
| 博學 | 3–4 格 | 1 次 | 較多 | 熟悉成語的玩家 |

計分由答對基本分、連續答對加成與完賽時間加成構成；錯誤嘗試會中斷連續答對，但不產生負分。使用提示會降低該字格得分。這使玩家可以完成每一局，同時保留追求高分的空間。

## 盤面與題組

- 首版採人工驗證的固定交錯題組，而非即時產生任意交叉盤，確保字格關係、難度與手機版面穩定。
- 每個題組定義盤面尺寸、成語起點、方向、答案與自撰提示。
- 交叉字格可以同時屬於橫向與直向成語，兩者答案必須一致；完成狀態由單一字格資料決定。
- 題組載入前執行結構驗證；無效題組不進入抽選池。若沒有可用題組，介面顯示可恢復的錯誤訊息並提供重新載入。
- 隨機抽題會避免立即重複上一局；最高分與偏好設定儲存在 localStorage，題組資料維持唯讀。

## 操作與狀態

- 滑鼠／觸控：點選空格，再點候選字；點已選字格可重新聚焦，但已答對字格不可修改。
- 鍵盤：Tab 移動至互動控制項；方向鍵在盤面空格間移動；Enter 選取；Esc 取消選取。
- 提示：有選取字格時揭示該格；未選取時揭示下一個未完成字格。
- 重設本局：清除本局答案、錯誤次數、分數與計時，保留難度與偏好。
- 頁面失焦不暫停計時；首版不提供暫停功能，避免背景狀態分歧。

遊戲狀態包含題組、難度、已填字格、目前選取字格、候選字、提示餘額、答對數、錯誤數、連續答對、分數、開始時間與完成時間。狀態更新由純函式處理，介面只負責渲染與派送玩家操作。

## 視覺設計

採用「昭和電視益智節目 × 和紙書房」：深墨藍舞台承載米白和紙字格，墨色文字配合朱砂紅正確狀態與低調黃銅資訊牌。整體保留實體棚景、印刷物與手工紙材的觸感，避免制式數位儀表板外觀。

可選但首版不採用的方向包括純和紙書房、活版印刷報刊、現代日式展覽。選定方向兼具電視節目張力、中文閱讀清晰度與遊戲狀態辨識度。

### 動畫與音效

- 正確：字格翻入並落下朱砂印，總長不超過 240 毫秒。
- 錯誤：字格水平位移一次，總長不超過 180 毫秒，不使用閃爍。
- 完成：題板依序亮起並顯示結算紙卡，不使用大量粒子。
- `prefers-reduced-motion: reduce` 時移除位移與翻轉，只保留顏色變化。
- 音效由 Web Audio API 合成短促木魚與印章聲；預設音量保守，可完全關閉。

## 響應式配置

- 手機（小於 640px）：單欄；狀態列、盤面、候選字與操作列依序排列，字格至少 44×44 CSS 像素。
- 平板（640–1023px）：盤面置中，狀態列與操作列分區排列。
- 桌機（1024px 以上）：左側主盤面，右側資訊與操作面板；限制內容最大寬度，避免盤面過度拉伸。
- 盤面超出可用寬度時等比縮放，不使用水平捲動作為主要操作方式。

## 無障礙與內容設計

- 所有互動控制項皆可使用鍵盤，焦點樣式清楚且不只依靠顏色傳達狀態。
- 字格提供可讀名稱，例如「第 2 列第 4 欄，尚未作答」；狀態更新透過適度的 `aria-live` 訊息通知。
- 文字與背景至少符合 WCAG 2.1 AA 一般文字對比度。
- 支援 200% 縮放、減少動態偏好與觸控操作。
- 中文介面使用臺灣用語，文件、註解與 UI 文案遵循專案 AGENTS.md 的名詞翻譯規範。

## 程式架構

- `index.html`：語意化頁面骨架、SEO Meta Tag、Open Graph 與結構化資料。
- `styles/`：設計變數、基礎排版、元件與響應式樣式。
- `scripts/data/`：題庫與題組資料。
- `scripts/game/`：題組驗證、盤面模型、遊戲狀態與計分純函式。
- `scripts/ui/`：畫面渲染、輸入控制、對話框與無障礙通知。
- `scripts/services/`：localStorage 偏好與 Web Audio API 音效。
- `scripts/main.js`：組合各模組並啟動應用程式。
- `tests/`：使用 Node.js 內建測試執行器驗證純 JavaScript 模組，不依賴大型測試框架。

頁面載入後，主程式驗證題組資料、讀取玩家偏好並顯示難度選擇。每次玩家操作轉換為明確動作，交由狀態函式產生下一狀態，再由渲染器更新必要的 DOM。儲存與音效失敗不得中斷遊戲；這類錯誤只降級功能並保留可玩性。

## 測試策略

- 單元測試：題組結構、交叉字一致性、難度過濾、候選字、填字結果、提示、計分、完成判定與偏好儲存降級。
- 瀏覽器手動測試：完整遊戲循環、重新開始、下一局、外部查詢連結、重新整理與 localStorage。
- 響應式測試：手機、平板、桌機三個斷點，包含觸控目標與 200% 縮放。
- 無障礙測試：鍵盤流程、焦點順序、可讀名稱、對比度、減少動態與螢幕閱讀器通知。
- 靜態部署測試：以本機 HTTP 伺服器與 GitHub Pages 子路徑載入，確認所有資源使用相對網址。

## 不在首版範圍

- 後端服務、帳號、排行榜同步與多人遊戲。
- 執行時擷取教育部網站或大量重製其釋義內容。
- 任意成語自動交叉盤生成器、玩家自建題組與題庫編輯器。
- 安裝型 PWA、離線快取、廣告、付費與金流。

## 驗收條件

1. 靜態 HTTP 伺服器開啟首頁後，可完成三種難度各至少一局。
2. 每盤包含多組橫直成語，交叉字一致，所有空格均可使用滑鼠、觸控與鍵盤完成。
3. 提示、計分、錯誤回饋、完成判定、結算與下一局均可正常運作。
4. 重新整理後保留最高分與偏好；localStorage 不可用時仍能遊玩。
5. Node.js 內建測試全部通過，且題庫驗證涵蓋所有內建題組。
6. 手機、平板與桌機斷點無遮擋或必要的水平捲動，主要觸控目標至少 44×44 CSS 像素。
7. 鍵盤可完成整局，減少動態設定有效，介面對比度符合 WCAG 2.1 AA。
8. README 與三份指定文件完整，並可依文件部署至 GitHub Pages。
9. Git 提交不包含憑證、token、`.env` 或私人金鑰。

## 風險與因應

- 題庫正確性：使用人工整理、資料結構驗證與全題庫測試；每題保留官方查詢入口。
- 交錯盤版面：首版採固定題組與明確尺寸，避免執行時生成不可解或過寬盤面。
- 中文字型載入：使用系統日系無襯線字型堆疊，不依賴遠端字型即可閱讀。
- localStorage／Web Audio API 不可用：捕捉錯誤並降級，不阻擋核心遊戲。
- Git 推送：只有在提交完成、檢查 remote、branch 與 commit 並向使用者顯示後，才執行 `git push`；若無 remote，停止並回報缺少的儲存庫資訊。
